# SCADA Diagram Editor

แยกออกมาจาก Wastewater Monitor Dashboard (`scada-monorepo-latest/artifacts/scada-dashboard`)
เป็นเครื่องมือเดี่ยว (standalone) สำหรับตั้งค่า:

- ตึก (Buildings)
- ถังบำบัด (Tanks) ต่อตึก
- เครื่องจักร (Equipment) ต่อตึก
- แผนผังกระบวนการบำบัด (Process Flow Diagram) — ฉาก 3D แบบ isometric ลาก-วางถัง/เครื่องจักร/เดินท่อ

ไม่มีระบบ auth/role — ทุกอย่างแก้ไขได้เสมอ เพราะเป็นเครื่องมือภายในสำหรับตั้งค่าเท่านั้น

## เมนูในแอป

| แท็บ | ทำอะไรได้บ้าง |
| --- | --- |
| **ตึก / Buildings** | เพิ่ม/แก้ไข/ลบตึก, จัดลำดับการแสดงผล, ตั้งค่าฐาน (กำลังการบำบัด, สถานะ, ค่า KPI ตั้งต้นเช่น pH/DO/ความขุ่น/คลอรีนคงเหลือ) |
| **ถังบำบัด / Tanks** | จัดการถังของแต่ละตึก — เพิ่ม/แก้ไข/ลบ/จัดลำดับ, ตั้งชนิดถัง (ปรับสภาพ, ไร้อากาศ, เติมอากาศ, ตกตะกอน, สัมผัสคลอรีน, เก็บสลัดจ์, กำหนดเอง), ความจุ และระดับน้ำตั้งต้น |
| **เครื่องจักร / Equipment** | จัดการเครื่องจักรของแต่ละตึก — เพิ่ม/แก้ไข/ลบ, ตั้งชนิด (ปั๊ม, เครื่องเป่าอากาศ, เครื่องตีน้ำ, บาร์สกรีน, ปั๊มจ่ายสาร, วาล์ว, เซ็นเซอร์ ฯลฯ), สถานะ (เดินเครื่อง/หยุด/ผิดพลาด), และผูกกับถังที่เกี่ยวข้อง |
| **กระบวนการ / Diagram** | แผนผังกระบวนการแบบ 3D — ลากถัง/เครื่องจักรจาก palette วางบนฉาก, ลาก/scroll หมุน-ซูมกล้อง (isometric), ลากวัตถุย้ายตำแหน่ง (snap เข้ากริดอัตโนมัติ), คลิกจุดต่อสีฟ้ารอบวัตถุ (เหนือ/ใต้/ออก/ตก) 2 จุดเพื่อเดินท่อ, ปุ่ม "จัดให้อัตโนมัติ" จัดวางใหม่ตามลำดับถัง, "บันทึก" เพื่อเซฟ layout |

ปุ่ม **"คืนค่าเริ่มต้นทั้งหมด / Reset"** มุมขวาบน (ทุกแท็บ) ล้างข้อมูลทั้งหมดกลับไปเป็น 4 ตึกตัวอย่างของศิริราช

## โมเดล 3D ที่ใช้ (`public/models/`)

ถัง/เครื่องจักรบางชนิดใช้โมเดล 3D จริง (`.glb`) แทนรูปทรงเรขาคณิตพื้นฐาน ส่วนที่เหลือ (ชนิดอื่นๆ) ยังเป็นทรงกระบอก/กล่องมาตรฐานที่สร้างด้วยโค้ด:

| ไฟล์ | แทนที่ชนิด | ขนาด |
| --- | --- | --- |
| `ket-nuoc-inox02.glb` | ถัง**ปรับสภาพ** (equalization) — ถังน้ำสแตนเลสทรงนอนบนฐานเหล็ก | ~270 KB |
| `water-septic-tank.glb` | ถัง**ไร้อากาศ** (anoxic) | ~550 KB |
| `daf-tank.glb` | ถัง**เติมอากาศ** (aeration) — ระบบ Dissolved Air Flotation (DAF) | **~61 MB** ⚠️ |
| `water-pump-2.glb` | **ปั๊ม** (pump) | ~4.5 MB |

โมเดลแต่ละไฟล์จะถูกจัดกึ่งกลาง/วางฐานติดพื้น/ปรับสเกลอัตโนมัติจาก bounding box จริงตอนโหลด (ดูใน
`Scada3DScene.tsx`) เพื่อให้จุดต่อท่อรอบตัวอยู่ที่ขอบจริงของโมเดลเสมอ ไม่ว่าโมเดลจะมีสัดส่วนแบบไหน

⚠️ **`daf-tank.glb` มีขนาดใหญ่ผิดปกติ (~61 MB)** เกินขนาดแนะนำของ GitHub (50 MB) — push ผ่านได้ปกติแต่
ควรพิจารณาบีบอัดด้วย Draco/texture compression (เช่นผ่าน `gltf-transform` CLI) ก่อน deploy จริงเพื่อลดเวลาโหลด
ไฟล์นี้ตั้งใจไม่ preload ล่วงหน้า จะโหลดเฉพาะตอนเปิดตึกที่มีถังเติมอากาศจริงๆ เท่านั้น

ถ้าจะเพิ่ม/เปลี่ยนโมเดล: วางไฟล์ `.glb` ไว้ใน `public/models/` แล้วอ้างอิง URL ผ่านฟังก์ชัน `modelUrl()` ใน
`Scada3DScene.tsx` (ห้าม hardcode path ขึ้นต้นด้วย `/` ตรงๆ เพราะจะพังตอน deploy ใต้ subpath เช่น GitHub Pages)

## การเก็บข้อมูล

ข้อมูลเก็บใน `localStorage` ของเบราว์เซอร์:

- `scada.buildings.v2` — รายชื่อตึก/ถัง/เครื่องจักรทั้งหมด
- `scada.flow.3d.v1.<buildingId>` — layout ของแผนผัง 3D ต่อตึก (ตำแหน่งวัตถุ, ท่อที่เชื่อมและด้านที่ต่อ)

**หมายเหตุ:** คีย์ `scada.flow.3d.v1.*` เป็น schema ใหม่สำหรับฉาก 3D (ตำแหน่งวัตถุ + ด้านที่ต่อท่อ) **ไม่ใช้ร่วมกับ
คีย์เดิม `scada.flow.v2.*`** ของ dashboard หลักอีกต่อไป เนื่องจากโครงสร้างข้อมูลเปลี่ยนไปคนละแบบ (2D → 3D)

## วิธีติดตั้ง

ต้องมี [Node.js](https://nodejs.org/) (เวอร์ชันเดียวกับที่ CI ใช้คือ 24 — ดูใน `.github/workflows/`) และ [pnpm](https://pnpm.io/) ก่อน

```bash
pnpm install   # หรือ npm install
pnpm dev       # หรือ npm run dev — เปิดที่ http://localhost:5173
```

### Build

```bash
pnpm build
```

Build output จะอยู่ที่ `dist/`. ถ้าต้อง deploy ใต้ subpath (เช่น GitHub Pages) ให้ตั้ง
`BASE_PATH=/ชื่อ-repo/` ตอน build เช่น:

```bash
BASE_PATH=/test-demoscadawwtp/ pnpm build
```

โปรเจกต์นี้มี GitHub Actions workflow (`.github/workflows/`) ตั้งค่า `BASE_PATH` ให้อัตโนมัติและ deploy ขึ้น
GitHub Pages ทุกครั้งที่ push เข้า `main`
